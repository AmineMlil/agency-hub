=== 1) Install Git (if `git` is not recognized) ===
Download: https://git-scm.com/download/win
During setup, choose "Git from the command line and also from 3rd-party software" so `git` works in PowerShell.
Restart the terminal after installing.

=== 2) Create the first commit (in PowerShell) ===
cd "c:\Users\amine\OneDrive\Documents\agency-hub"

git init
git branch -M main
git add -A
git status

git commit -m "Initial commit: Agency Hub"
# If Git asks who you are, run once (use your real email):
# git config --global user.name "Your Name"
# git config --global user.email "you@example.com"
# Then run the commit again.

=== 3) Create an empty repo on GitHub ===
- Open https://github.com/new
- Repository name: e.g. agency-hub
- Leave it EMPTY (no README, no .gitignore from GitHub)
- Create repository

=== 4) Connect and push ===
Replace YOUR_USER and agency-hub with your GitHub username and repo name.

git remote add origin https://github.com/YOUR_USER/agency-hub.git
git push -u origin main

(GitHub may ask you to log in via browser or a personal access token.)

=== 5) Import into Vercel ===
- https://vercel.com/new → Import Git Repository → pick this repo
- Framework: Next.js (auto)
- Add Environment Variables in Vercel (Project → Settings → Environment Variables):
  - AUTH_SECRET = long random string (16+ chars)
  - DATABASE_URL = your database URL (see note below)

NOTE FOR VERCEL:
SQLite files do not work well on Vercel serverless hosting. For a real deployed app,
use PostgreSQL (e.g. Neon https://neon.tech ) and point DATABASE_URL there, then run:
  npx prisma db push
against that Postgres URL from your PC (or use Vercel build command with migrate).

For a quick "does it build" test on Vercel, you can still import the repo; fixing the
database for production is the next step.
